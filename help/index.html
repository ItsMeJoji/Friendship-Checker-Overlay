2. Sign in to Twitch

The first time you open the overlay, it sends you to Twitch so you can approve access. The app requests the scopes it needs to read chat and channel point redemptions.

Approve the Twitch prompt when it asks for access. After you authorize, the overlay stores the token locally in the browser so you do not have to log in again on the same machine.

3. Use the OBS-ready overlay URL

After you sign in, press F12 to open DevTools and find the browser-source URL in the console with the access token already attached. Use that final URL in OBS or in whatever tool is loading the overlay.

Step flow: 1. Open the overlay with your `username` and `client_id`. 2. Let Twitch authenticate the account. 3. Press F12, open the Console Tab, and copy the final URL the app prints after login. 4. Paste that URL into OBS as a browser source. 5. Keep the browser source on the same machine/browser profile that authorized Twitch.

What the account needs

The Twitch account should be the broadcaster account for the channel you are connecting. That account must allow the overlay to read chat and listen for channel point redemptions.

  • Broadcaster login: use the account tied to the live channel.
  • Correct Client ID: this must match the Twitch app you created.
  • Same browser profile: local token storage stays with that browser/device.

Troubleshooting

  • Blank overlay: double-check the `username` and `client_id` values in the URL.
  • Repeated login prompts: clear old local storage for the site and authorize again.
  • No redemptions: confirm the channel point reward exists and matches the expected title in the overlay code.
  • Wrong channel: make sure the Twitch account and channel name belong to the same broadcaster.
